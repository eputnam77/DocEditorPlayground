Advisory Circular AC 20-115D

Subject: Airborne Software Assurance

Purpose: This advisory circular (AC) provides guidance for the development of airborne software in accordance with RTCA/DO-178C.

1. Introduction

This AC describes acceptable means, but not the only means, for demonstrating compliance with the applicable airworthiness regulations for the software aspects of airborne systems and equipment certification.

Note: The guidance in this AC is not mandatory and does not constitute a regulation.

2. Applicability

  • All new software development for airborne systems
  • Software modifications and upgrades
  • Legacy software re-certification

3. Definitions

  1. Software Life Cycle: The sequence of processes through which software passes from conception to retirement.
  2. Verification: The evaluation of software to determine whether products of a given development phase satisfy the conditions imposed at the start of that phase.

4. Guidance

Applicants should:

  • Establish a software development plan
  • Document requirements, design, code, and test results
  • Perform configuration management and quality assurance
  • Conduct reviews and audits at key milestones
// Example of a software life cycle process
Requirements → Design → Code → Test → Release

5. References

  • FAA Order 8110.49, Software Approval Guidelines

End of Advisory Circular
